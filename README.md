1.波浪形

redis:用户记录:{
  充值金额
  掉落28分组
  掉落走势模型(普通模式，高涨跌)
  基于当前充值盈利
  当前打码量
  上一次的RTP模式(扶/杀/其他)
  跌落至多少比例开始上浮
  上浮最高上限
  基于这次充值打码上限开始慢杀
  我的rtp列表/如果是6生肖(11-30)/其他游戏(1-7) 基于28定律再单独细分 
  是否触发底部反弹扶植
  已经扶植金额
  最大底部扶植次数
  当前已经触发次数
}




流程
用户充值->激活redis缓存，随机掷骰子分配这一次的2-8定律，判断她这一次的充值金额是否大于(触发高涨跌配置金额)
当前充值盈利设置0，上一次的rtp设置为0，算出跌落百分比开始扶，算出最大打码量扶，分配rtp列表为[]

用户进入游戏——>通过游戏ID获取是什么类型的rtp，再基于28分组获取细化的rtp，随机投掷给列表中的随机rtp

用户投注->判断当前打码有没有上限是否需要慢杀，判断全局水位是否需要必杀，判断是否已经达到触底(判断扶植次数是否上限，扶植>1随机50%下降趋势判断是否扶植) 
开始扶植-判断当前盈利
